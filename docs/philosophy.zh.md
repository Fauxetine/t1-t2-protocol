# 设计理念

## 问题：自指盲区

当一个 AI 模型检查自己的输出时，它使用同一套训练数据、同一套推理偏好、同一套系统性偏差。这种自指验证有一个根本局限：**一个模型无法发现自己看不到的东西。**

这与智能水平无关。更聪明的模型并不能更好地发现自己的盲区——它只是有不同的盲区。因此，自我反思作为验证策略是不够的。

## 解法：异构验证

T2 协议引入**异构验证**：评估者应该是和产出者不同的模型。两个在不同数据上训练、具有不同架构和不同推理偏好的模型，会有不同的盲区。一个模型的弱项可能是另一个模型的强项。

这借鉴了成熟领域的同行评审制度：
- 学术论文不由作者自己评审
- 代码不由开发者自己审查
- 财务审计不由被审计方自己做

AI 验证也应遵循同样的原则。

### 异构 ≠ 更强

评估模型不需要比产出模型"更聪明"。它只需要**不同**。价值来自分布差异，而非能力优势。一个训练数据不同的 7B 模型，可以捕捉到 70B 模型遗漏的盲区。

## 安全-智能悖论

当前 LLM 的安全依赖于依赖模型本身的对齐技术：RLHF、系统提示词、输出过滤。这造成了一个悖论：

**随着模型变得更加智能，它们变得更能绕过依赖自身推理的安全机制。**

使模型有用的同一种能力，也使它们更难被约束。

## 解法：确定性安全

Checksum 和协议的结构验证是**确定性的**。它们使用纯正则表达式和字符串操作——零 LLM 依赖。一个模型无法"通过思考绕过"一个正则规则，无论它变得多智能。

这是核心架构洞见：**智能可以无限增长，但安全必须扎根于不增长的代码。**

## 结构化推理

T1 协议解决另一个问题：模糊的问题产生不可靠的答案。在评估答案是否正确之前，我们必须确保问题是被明确定义的。

四个层级（L1-L4）遵循科学方法：

| 层级 | 功能 | 科学类比 |
|------|------|---------|
| **L1 事实** | 可验证的、可观测的真相 | 数据 |
| **L2 假设** | 需要验证的合理前提 | 工作假设 |
| **L3 假说** | 可证伪的主张，附带证据追踪 | 可检验的预测 |
| **L4 未知** | 推理的已知边界 | 已知的未知 |

分解为这些层级的问题可以被系统地回答。未经分解的问题则不能。

## 与 AGI 安全的关系

如果通用人工智能出现，两个问题变得紧迫：

1. **如何让 AGI 的决策可理解？** — T1 的层级分解为复杂推理链提供了可追溯框架。

2. **如何确保 AGI 的行为安全？** — 确定性检查（checksum、结构验证）提供了一个不受智能水平影响的安全层。"智能"和"信任"的分离不是一种便利——它可能是一种必要。

## 本协议不做的事

- **不调用工具** — 那是其他 MCP 服务器的角色
- **不存储数据** — 不是协议层的责任
- **不对模型进行基准测试** — 这不是一个基准测试
- **不产生答案** — 验证不是生成

## 开放问题

- **最优异构度**：两个模型应该相差多大才能最大化盲区覆盖？同系列？不同系列？不同模态？
- **递归边界**：交叉验证的边际收益在多少层后趋近于零？当前启发式在深度 ≥ 3 时终止。
- **权重校准**：不同领域的评估标准权重应如何校准？当前由用户指定；未来可以学习。
