# 全链路示例：T1 → 决策 → T2

展示完整的异构验证工作流程。

## 步骤 1：T1 — 结构化问题

```
调用：t1_protocol("评估微服务迁移可行性")
  → 返回 L1/L2/L3/L4 结构化提示词
```

## 步骤 2：基于结构化信息做决策

```
"先试点迁移 1-2 个服务，验证服务边界清晰度和团队能力。
如果 3 个月内试点成功，再逐步迁移其余服务。"
```

## 步骤 3：T2 — 交叉验证决策

```
调用：t2_protocol(决策文本)
  → 返回：

置信度：中偏高
  - 试点策略是低风险的合理路径
  - 建议：补充量化验收标准

| 采纳 | 内容 | 修正 | 证据层级 |
|------|------|------|----------|
| ✅ | 试点迁移策略 | — | L1 |
| ✅ | 3 个月评估周期 | — | L2 |
| ⚠️ | 未定义"成功"标准 | 补充指标（部署频率提升 X%，故障率不增加） | L2 |
```

## 价值链

```
模糊问题 → T1 结构化分解 → 基于事实和假设做决策，而非直觉
决策 → T2 异构验证 → 发现盲区并修正
修正后的决策 → 执行 → 更可靠的结果
```

这就是完整的 T1/T2 协作流程。
