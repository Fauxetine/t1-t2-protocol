# T2 示例：交叉验证一个决策

## 场景

你已经做了一个决策，想让另一个模型检查盲区。

## 输入

调用 `t2_protocol`，传入：

```
我决定采用微服务架构：
1. 提高部署频率
2. 团队可以独立开发
3. 技术栈可以多样化
```

## 输出

```
置信度：中偏低
  - 描述：方向合理，但关键约束缺乏验证
  - 递归深度：Layer 0
  - 权重假设：事实优先

采纳建议：

| 采纳 | 内容 | 修正 | 证据层级 |
|------|------|------|----------|
| ✅ 采纳 | 微服务提高部署频率 | — | L1 |
| ⚠️ 保留 | 团队独立开发 | 先验证服务边界清晰度 | L2 |
| ⚠️ 保留 | 技术栈多样化 | 5人团队运维多技术栈有难度 | L2 |
| ❌ 不适用 | 未考虑分布式复杂度 | 补充运维和监控方案 | L2 |
```

## 价值

- 发现自我反思遗漏的盲区
- 按证据层级标注置信度（L1=已验证，L2=需处理）
- 提供可操作的修正建议，而非单纯批评
